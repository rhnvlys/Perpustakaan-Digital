# Product

## Register

product

## Users

Mahasiswa menggunakan aplikasi untuk masuk ke akun, melihat ringkasan peminjaman, mencari buku, mengajukan peminjaman, memantau status pengajuan, melihat pinjaman aktif, dan mengelola profil.

Admin perpustakaan menggunakan aplikasi untuk memantau ringkasan koleksi, menyetujui atau menolak pengajuan peminjaman, mengelola data buku, dan melihat data peminjaman.

## Product Purpose

Perpustakaan Digital adalah antarmuka web untuk sistem informasi perpustakaan kampus. Keberhasilan produk berarti mahasiswa dapat menemukan dan mengajukan peminjaman buku dengan cepat, sementara admin dapat mengelola koleksi dan pengajuan tanpa kebingungan.

## Brand Personality

Akademik, jelas, dan ramah. Antarmuka harus terasa tepercaya untuk tugas kampus, tetapi tetap ringan dipakai di layar mobile.

## Anti-references

Jangan terlihat seperti halaman placeholder "Hello World", landing page generik, dashboard terlalu gelap, atau UI yang penuh dekorasi tanpa membantu tugas. Hindari pola yang tidak sesuai prototype Figma, terutama warna aksen selain teal-cyan utama dan komponen yang tidak konsisten antar layar.

## Design Principles

1. Dahulukan alur tugas mahasiswa dan admin daripada dekorasi.
2. Ikuti kosakata komponen Figma secara konsisten: header mobile, sidebar, kartu putih, input abu terang, badge status, dan tombol gradient teal.
3. Buat state UI nyata walaupun tanpa backend: navigasi, filter, pencarian, login demo, pengajuan, persetujuan, dan empty state harus terasa bekerja.
4. Pertahankan mobile-first sebagai pengalaman utama, lalu beri pembingkai desktop yang rapi.
5. Gunakan motion singkat hanya untuk feedback, transisi layar, dan sidebar.

## Accessibility & Inclusion

Target minimal WCAG AA untuk kontras teks, fokus keyboard yang terlihat, label form eksplisit, struktur HTML semantik, dan dukungan `prefers-reduced-motion`. Semua fungsi utama harus bisa digunakan tanpa hanya mengandalkan warna.
